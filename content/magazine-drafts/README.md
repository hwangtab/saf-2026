## 작성 원칙

- **톤**: 존칭 본문. 감정적 과장 없이, 데이터·관찰·1인칭 경험을 통해 설득.
- **캠페인 프레이밍 준수** (CLAUDE.md):
  - 출품 작가 127명 = 연대자. "금융 피해 당사자"로 묘사 금지.
  - 금융 차별 데이터는 한국 예술인 전체 문제.
  - "작품 판매 → 상호부조 기금 → 저금리 대출" 흐름 강조.
- **SEO**: 각 글마다 target keyword, meta description, 내부링크 3개 이상 명시.
- **분량**: 1,800~3,500자. FAQ 섹션 포함 시 추가.
- **중복 금지**: 신규 글 작성 전 Supabase `stories` 테이블에서 slug/주제 검색 필수. 107편이 이미 발행되어 있음.

## 이미지·작품 링크 규칙

모든 원고에 실제 씨앗페 작품 이미지가 2~3장 삽입되어 있으며, 본문에 `/artworks/{uuid}` 링크로 작품이 참조됩니다. 포맷은 [app/\[locale\]/stories/\[slug\]/page.tsx](../../app/[locale]/stories/[slug]/page.tsx)의 `extractFirstImage`·`extractArtworkIds` 동작을 그대로 활용합니다.

```markdown
[![작가명, 〈작품명〉, 재료·크기](이미지URL)](/artworks/{uuid})
```

- **첫 이미지가 자동 썸네일**: 스토리에 `thumbnail` 필드가 비어 있으면 본문 마크다운의 **첫 이미지**가 OG·스키마·카드 썸네일로 사용됨
- **alt 텍스트 = 캡션**: `MarkdownRenderer`가 alt를 그대로 `<figcaption>`으로 렌더
- **작품 링크 자동 카드화**: 본문 중 `/artworks/{uuid}` 참조는 상세 페이지 하단 "이 글에서 소개한 작품" 섹션에 자동 노출
- **클릭 가능한 figure**: `[![alt](img)](/artworks/id)` 조합은 이미지 전체가 작품 페이지로 이동 + "작품 보러 가기 →" 호버 오버레이
- **FAQ → FAQPage schema 자동 생성**: 본문에 `**Q. ...?**` / `A. ...` 패턴이 있으면 `lib/utils/extract-faqs.ts`가 추출해 JSON-LD로 자동 출력 (SERP FAQ 리치 결과용)
