# Part 15. 검색 인지 — 진입로 설계와 시스템 사양

_씨앗페 사이트의 Google·네이버 검색 진입로 설계 매뉴얼_

**작성일**: 2026년 5월 · **최종 업데이트**: 2026년 5월
**대상**: 사이트를 새로 만들거나 갱신할 때 직접 참조하는 운영·개발 매뉴얼
**범위**: saf2026.com 사이트 자체의 검색 최적화. 네이버 블로그·카페·지식iN 등 네이버 생태계 채널 운영은 본 매뉴얼 범위 밖(통합매뉴얼 0.2의 1번 외부 유입·홍보 매뉴얼에서 다룸).

## 15.1 검색 매뉴얼의 본질 — 두 층의 언어

통합매뉴얼은 페르소나 B의 누적 신뢰 형성에 집중한 본질 카피("내 맘에 드는 그림을 사고, 그 힘으로 작가에게 불가능했던 대출을 가능케 합니다", "예술인 상호부조 플랫폼", "접근 가능한 격조", "자긍심 박스")를 정의했다. 이 카피는 사이트에 도착한 페르소나 B에게는 정합하지만, **사이트로 사람을 데려오는 검색 진입로는 만들지 못한다.** 사람들이 검색창에 치는 말은 "한국 작가 그림", "30만원 그림", "오윤 작품 가격", "처음 그림 사기" 같은 표층 언어다.

본 매뉴얼은 이 **두 층의 언어를 한 페이지 안에서 공존시키는 이중 구조**를 정의한다.

### 15.1.1 두 층의 정의

> **본질 카피와 검색 카피의 공존 [공통]**
>
> - **본질 카피** — 페르소나 B의 누적 신뢰 형성용. 보호 영역. 변경 권한은 이사장님.
>   - Hero 카피(Part 6.2의 28자 본문), 자긍심 박스 카피(Part 8), About 정체성 본문(Part 6.4), Sub-카피(Part 6.3)
> - **검색 카피** — 검색 진입로 형성용. 노출 영역. 운영팀 자율(본질 영역 침해 없는 한).
>   - 메타 타이틀·메타 디스크립션, OG 태그, 구조화 데이터, 이미지 alt 텍스트, 카테고리 페이지 설명, 매거진 글 제목·H1, 작품 카드 보조 텍스트, 작가 페이지 약력 설명, anchor text, 푸터 링크 텍스트

### 15.1.2 공존의 기본 규칙

같은 페이지 안에서 본질 카피와 검색 카피는 시각적으로 공존하되 의도가 다르다. 한 페이지의 사례(메인페이지 Hero·About 영역).

```
[시각 우선순위 1순위] Hero 큰 작품 이미지 — 거장 작품 + 가격 (Part 6.4 정합)
[시각 우선순위 2순위] Hero 카피 (본질) — "내 맘에 드는 그림을 사고, 그 힘으로..."
[시각 우선순위 3순위] Sub-카피 (본질) — "작품 거래의 10%가 354건의 안전망..."

[HTML 구조 H1] — "씨앗페 — 한국 현대미술 작가 112명의 원본 작품 348점 직매"
    (시각적으로 작게 표시되거나 About 영역과 통합. 검색 카피.)

[About 영역] 본질 카피 본문 + 검색 카피 보조 문구
    본문(본질) — "한국 작가 112명, 작품 365점" / "예술인 상호부조 플랫폼"
    보조(검색) — "한국 현대미술 작가 112명의 회화·판화·사진·조각 원본을 직매로 만나보세요"
```

본질 카피의 28자·5줄 보호는 그대로다. 검색 카피는 그 옆·아래에 자연스럽게 자리 잡되, 본질 카피를 가리거나 압도하지 않는다.

### 15.1.3 한국어 자연 어순 원칙 [공통, 네이버 비중 강함]

네이버의 자연어 처리는 한국어 어순·조사·관용 표현에 매우 민감하다. 같은 의미라도 어순이 어색하면 검색어와의 매칭이 약해진다.

> **검색 카피 작성의 한국어 자연 어순 원칙**
>
> - 자연: "한국 작가 그림 직매" / "30만원 이하 원본 작품" / "한국 현대미술 거장 작품 보기"
> - 부자연: "그림 작가 한국 직매" / "원본 작품 30만원 이하의" / "거장 한국 현대미술 작품 보기"
> - 검색어는 사람이 말하는 어순으로. 키워드 나열 회피.
> - 조사("의·을·를·에서") 자연스럽게 사용. 너무 많이 빼지 않음.

### 15.1.4 충돌 시 처리

- 본질 카피와 검색 카피가 같은 자리를 다툴 경우 → 본질 카피 우선, 검색 카피는 인접 위치로 이동
- 검색 데이터가 본질 카피 변경을 권유할 경우 → 통합매뉴얼 14.1 충돌 보고 시스템, 자동 변경 금지
- 검색 카피만 단독으로 다듬는 경우 → 운영팀 자율 (본질 카피 톤·격조와 정합하는 선에서)

## 15.2 검색 노출의 천장 — 명시적 인정

매뉴얼이 약속할 수 있는 천장을 명시한다. 천장 너머는 본 매뉴얼이 다루지 않는다.

> **검색 노출의 도달 가능 범위 [공통]**
>
> | 검색엔진 | 도달 가능 범위                                                                                                                         | 도달 불가 범위                                                                                                     |
> | -------- | -------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------ |
> | Google   | 브랜드·작가명·작품명·매체·가격대·컬렉팅 정보 등 거의 모든 검색 유형 상위 노출                                                          | 없음 (사이트 자체 최적화로 도달 가능 최대치)                                                                       |
> | 네이버   | 브랜드성·구체성 검색(예: "씨앗페", "씨앗페 오윤") 1위. 작가명·작품 long-tail("오윤 사후판화 가격") 외부 사이트로서 받을 수 있는 최대치 | Generic intent 검색("그림 사기", "30만원 그림") 사실상 비노출. 네이버 자사 생태계(블로그·카페·쇼핑) 우선 노출 구조 |

천장 너머를 노리려면 네이버 자체 채널의 콘텐츠 자산이 필요하다 — 이는 통합매뉴얼 0.2의 1번 매뉴얼 영역이며, **본 매뉴얼은 그 채널이 만들어 줄 백링크·인용을 사이트가 받을 준비를 하는 데까지만** 다룬다.

## 15.3 키워드 매핑 — 페르소나 4단계 × 검색 의도

페르소나 4단계 × 검색 의도(Intent) 매트릭스로 키워드 군과 답하는 페이지를 매핑한다.

### 15.3.1 4 × 4 키워드 매핑 표 [공통]

| 페르소나 단계     | 검색 의도     | 키워드 예시                                         | 답하는 페이지                        | Google 가능 | 네이버 가능 |
| ----------------- | ------------- | --------------------------------------------------- | ------------------------------------ | ----------- | ----------- |
| 1단 첫 구매자     | Commercial    | "30만원 그림", "처음 그림 사기", "저렴한 작가 그림" | /collections/under-300000            | 높음        | 낮음~중간   |
| 1단 첫 구매자     | Informational | "그림 컬렉팅 입문", "처음 그림 사는 법"             | 매거진 컬렉팅 가이드                 | 높음        | 중간        |
| 2단 재방문 컬렉터 | Transactional | "한국 작가 판화 직매", "원본 회화 구매"             | /collections/판화, /collections/회화 | 높음        | 낮음        |
| 2단 재방문 컬렉터 | Commercial    | "한국 작가 작품 가격대"                             | /artworks (필터)                     | 중간        | 낮음        |
| 3단 누적 컬렉터   | Navigational  | "오윤 작품", "박생광 그림", "이철수 판화"           | /artists/{작가}                      | 중간        | 높음        |
| 3단 누적 컬렉터   | Informational | "한국 민중미술 작가", "한국 현대미술 거장"          | 매거진 미술 산책                     | 중간        | 중간        |
| 4단 코어 컬렉터   | Informational | "오윤 사후판화 진품 보증", "박생광 드로잉 가치"     | 매거진 작가 이야기                   | 중간        | 중간        |
| 모든 단계         | Brand         | "씨앗페", "한국스마트협동조합", "saf2026"           | 메인페이지·About                     | 높음        | 높음        |

### 15.3.2 키워드 매핑의 운영 원칙

- 검색량이 큰 키워드가 우선 아님 — **페르소나 B 정합 + 외부 사이트 노출 가능성** 기준
- "그림 사기" 같은 generic transactional 검색은 매핑 표에서 제외 — 네이버 비노출, Google에서도 경쟁 매우 큼
- 신규 매거진 글·카테고리 페이지 생성 시 이 표를 갱신
- 매월 Search Console·서치어드바이저 데이터로 새로 발견되는 검색어는 노출 영역의 카피 다듬는 근거로 활용 (본질 변경 근거 아님 — 15.20 참조)

## 15.4 URL·슬러그 구조

### 15.4.1 URL 표준 [공통]

> **URL 슬러그 규칙**
>
> - 영문 슬러그 + 한글 표시명 구조
> - 소문자, 하이픈(-) 구분, 특수문자·공백 회피
> - URL 길이 60자 이하 권장
> - 한글 URL은 외부 공유 시 `%EC%98%A4...`로 깨지므로 회피
> - URL은 영구 — 변경 시 301 리다이렉트 필수

| 페이지 유형              | URL 표준                                   | 예시                                 |
| ------------------------ | ------------------------------------------ | ------------------------------------ |
| 메인                     | `/`                                        | `/`                                  |
| 작가 페이지              | `/artists/{영문-슬러그}`                   | `/artists/oh-yoon`                   |
| 작품 페이지              | `/artworks/{작품-id}` (현행 UUID 유지)     | `/artworks/4635c2c5-...`             |
| 카테고리 (상시)          | `/collections/{영문-슬러그}`               | `/collections/masters`               |
| 카테고리 (월별 아카이브) | `/collections/{영문-슬러그}/{YYYY-MM}`     | `/collections/curator-pick/2026-05`  |
| 매거진 글                | `/stories/{영문-슬러그}`                   | `/stories/oh-yoon-posthumous-prints` |
| 매거진 카테고리          | `/stories?category={카테고리}` (현행 유지) | `/stories?category=artist-story`     |
| 정책·정보                | `/{영문-슬러그}`                           | `/about`, `/faq`, `/transparency`    |

### 15.4.2 작품 페이지 URL의 보완 [공통]

현행 작품 URL은 UUID만으로 구성된다. UUID는 SEO 측면에서 검색어 매칭에 기여하지 않으므로, **메타 타이틀·H1·구조화 데이터·이미지 alt에 작가명·작품명을 풍부하게 담는 것**으로 보완한다 (15.7·15.8 참조). URL 자체의 변경은 권하지 않음 — 기존 백링크·공유 링크 보존이 우선.

### 15.4.3 쿼리 파라미터 처리

- 정렬·필터(`?sort=price-asc`, `?medium=printing`)는 canonical로 메인 카테고리 URL 지정
- `robots.txt`에서 `Disallow: /*?sort=*` 등 일부 차단 검토 (운영팀 + 개발 협의)

### 15.4.4 변경 시 절차

URL 변경은 통합매뉴얼 14.1 본질 영역에 준한다. 변경 시 다음 절차.

1. 301 리다이렉트 설정 (구 URL → 신 URL)
2. Search Console·서치어드바이저에 신 URL 색인 요청
3. 내부 링크 일괄 갱신
4. 백링크 보유 외부 사이트 중 영향 큰 곳에 갱신 요청

## 15.5 카테고리 페이지 SEO 아키텍처

통합매뉴얼 Part 6의 메인페이지 13개 영역 중 큐레이션 영역(D·E·G·H·I·J)을 **독립 카테고리 페이지로 전환**한다. 메인페이지는 미리보기 허브, 카테고리 페이지는 검색 진입로.

### 15.5.1 카테고리 페이지 구조 [공통]

| 메인 영역              | 독립 페이지 URL                       | H1 (검색 카피)                                                     | 갱신                             |
| ---------------------- | ------------------------------------- | ------------------------------------------------------------------ | -------------------------------- |
| [D] 이달의 큐레이터 픽 | `/collections/curator-pick/{YYYY-MM}` | "{년월} 이달의 큐레이터 픽 — 한국 작가 6점 큐레이션"               | 매월 신규 페이지, 옛 페이지 보존 |
| [E] 거장 컬렉션        | `/collections/masters`                | "한국 현대미술 거장 6인 — 오윤·박생광·신학철·민정기·이철수·박불똥" | 상시                             |
| [G] 신진 작가          | `/collections/emerging`               | "신진 작가 발견 — 한국 현대미술의 다음 세대"                       | 분기 갱신                        |
| [H] 30만원 이하        | `/collections/under-300000`           | "30만원 이하 첫 그림 — 한국 작가 원본 작품"                        | 상시                             |
| [I] 이달의 인기        | `/collections/popular/{YYYY-MM}`      | "{년월} 인기 작품 — 컬렉터가 주목한 한국 작가 작품"                | 매월                             |
| [J] 시즌 큐레이션      | `/collections/season/{YYYY-MM}`       | "{년월} 시즌 큐레이션 — {시즌 주제}"                               | 매월                             |

### 15.5.2 큐레이션 아카이브화 — 콘텐츠 자산 누적의 핵심 [공통]

> **매월 갱신 = 옛 콘텐츠 보존**
>
> - 큐레이터 픽이 매월 덮어쓰여지면 새 콘텐츠가 옛 콘텐츠를 지운다 — SEO 자산 손실
> - URL에 `{YYYY-MM}`을 포함시켜 매월 신규 페이지를 생성하면 12개월 후 12개, 24개월 후 24개의 검색 진입로가 누적
> - 옛 페이지는 noindex 처리하지 않음 — long-tail 검색 진입로로 작동
> - 메인페이지 [D] 영역은 항상 최신 월 페이지를 미리보기로 보여줌

### 15.5.3 허브-스포크 구조

- **허브**: 메인페이지(`/`) → 카테고리 페이지 6개 + 작가 페이지 16명 + 매거진 카테고리 3개로 분기
- **스포크**: 각 카테고리 페이지 → 개별 작품 페이지로 분기
- **잎**: 작품 페이지 → 같은 작가·매체·가격대 작품 추천(내부 링크)

이 구조는 통합매뉴얼 Part 6의 13개 영역과 자연 정합한다 — 13개 영역의 미리보기 흐름이 검색 친화 카테고리 구조로 그대로 번역된다.

### 15.5.4 카테고리 페이지의 필수 요소

각 카테고리 페이지는 다음을 포함한다.

- H1 (검색 카피, 한국어 자연 어순)
- 카테고리 설명문 (150~300자, 검색 카피 자연 노출)
- 작품 카드 그리드 (3~24점, 페이지네이션)
- 구조화 데이터: `ItemList` + `BreadcrumbList`
- 메타 타이틀·디스크립션 표준 (15.7 참조)
- 푸터에 다른 카테고리 링크 5개

## 15.6 구조화 데이터 (JSON-LD)

페이지 유형별 schema.org 마크업. Google이 주요 활용, 네이버도 일부 인식(OG·Article 등). **JSON-LD 형식 사용**, microdata·RDFa는 회피.

### 15.6.1 페이지 유형별 schema 매핑 [Google 중심, 네이버 부수 효과]

| 페이지 유형     | 필수 schema                   | 권장 schema             |
| --------------- | ----------------------------- | ----------------------- |

| 메인 페이지     | `Organization` + `WebSite`    | `SearchAction`          |
| FAQ 페이지/영역 | `FAQPage`                     | —                       |
| 오프라인 전시   | `Event`                       | `Place`                 |

## 15.7 메타데이터 표준

페이지 유형별 메타 타이틀·디스크립션·OG 작성 규칙. **OG 태그는 네이버 검색 결과 카드에도 사용되므로 양쪽 공통 자산**.

### 15.7.1 페이지 유형별 표준 [공통]

| 페이지          | 메타 타이틀 (50~60자)                                      | 메타 디스크립션 (120~155자)                                                                                                            |
| --------------- | ---------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| 메인            | "씨앗페 — 한국 작가 112명 원본 작품 348점 직매"            | "한국 현대미술 작가 112명의 회화·판화·사진·조각 원본 348점. 구매 354건이 예술인 저금리 대출이 된 상호부조 플랫폼. 무료 배송·7일 반품." |
| 작품            | "{작품명} — {작가명} {매체} {크기} \| 씨앗페"              | "{작가명}의 {작품명}({매체}, {크기}, {연도}). {가격} 원본 작품. 한국스마트협동조합 직매·무료 배송."                                    |
| 작가            | "{작가명}({생몰년도}) — 한국 현대미술 작가 작품 \| 씨앗페" | "{작가명} 대표 작품과 약력. {매체} 중심의 한국 {분야} 거장/작가. 씨앗페에서 원본 작품 직매·진품 보증."                                 |
| 카테고리 (상시) | "{카테고리명} — 한국 작가 {특성} 작품 \| 씨앗페"           | "{카테고리 설명, 작품 수, USP}. 한국 작가 원본·작가 직매·무료 배송."                                                                   |
| 카테고리 (월별) | "{YYYY-MM} {카테고리명} — 한국 작가 작품 \| 씨앗페"        | "{년월}의 {카테고리 주제, 작품 수}. 한국 작가 원본·작가 직매."                                                                         |
| 매거진 글       | "{글 제목} — {카테고리} \| 씨앗페 매거진"                  | "{글 핵심 내용 1~2문장}. 씨앗페 매거진 — 작가 이야기·컬렉팅 가이드·미술 산책."                                                         |
| FAQ             | "자주 묻는 질문 — 씨앗페 작품 구매·상호부조 안내"          | "씨앗페 작품 구매·배송·환불·조합원 가입·상호부조 대출 등 자주 묻는 질문. 한국스마트협동조합."                                          |

### 15.7.2 메타 작성 원칙

- **타이틀**: 핵심 키워드는 앞 30자 안에. 브랜드명은 끝에 ` | 씨앗페` 또는 ` - 씨앗페`로 통일
- **디스크립션**: 구체 숫자(348점·112명·354건·95%) 적극 활용. 검색 결과에 USP를 드러냄
- **자연 어순**: 키워드 나열 회피. 한국어 자연 어순 (15.1.3)
- **중복 회피**: 같은 메타가 여러 페이지에 중복되지 않도록. 카테고리 페이지마다 고유 디스크립션 필수

### 15.7.3 OG 이미지 표준 [공통]

- 크기: 1200×630px
- 형식: JPG 또는 WebP
- 페이지 유형별 OG 이미지:
  - 메인: 사이트 대표 OG 이미지 (현행 유지)
  - 작품: 작품 이미지 + 작가명·작품명 텍스트 오버레이
  - 작가: 작가 대표작 이미지 + 작가명·생몰년도
  - 카테고리: 카테고리 대표 작품 콜라주 + 카테고리명
  - 매거진: 글 대표 이미지 + 글 제목

## 15.8 이미지·alt·파일명 규칙

미술품 사이트의 검색 자산 중 가장 비중이 큰 영역. Google 이미지 검색 + AI 답변 인용 + 네이버 이미지 검색 모두에 영향.

### 15.8.1 파일명 표준 [공통]

> **이미지 파일명 규칙**
>
> - `{artist-slug}-{work-slug}-{medium}-{year}.{ext}`
> - 영문 슬러그, 소문자, 하이픈 구분
> - 예: `oh-yoon-kalnore-woodcut-1985.webp`
> - 회피: `IMG_3421.jpg`, `image-1.jpg`, `한글파일명.jpg`

### 15.8.2 alt 텍스트 표준 [공통]

| 이미지 유형        | alt 표준                                     | 예시                                                    |
| ------------------ | -------------------------------------------- | ------------------------------------------------------- |
| 작품 메인 이미지   | "{작가명}, {작품명}, {매체}, {크기}, {연도}" | "오윤, 칼노래, 목판화, 50x60cm, 1985"                   |
| 작품 디테일 이미지 | "{작가명}, {작품명} {부분 설명}"             | "오윤, 칼노래 화면 우측 부분 디테일"                    |
| 작가 초상          | "{작가명}({생몰년도}) 초상"                  | "오윤(1946-1986) 초상"                                  |
| 카테고리 헤더      | "{카테고리명} 컬렉션 대표 작품"              | "한국 현대미술 거장 컬렉션 대표 작품"                   |
| 매거진 이미지      | 글의 내용을 1~2문장으로                      | "오윤의 1985년 목판화 칼노래 — 작품의 사후 인쇄본 비교" |
| 로고               | "씨앗페 로고"                                | "씨앗페 로고"                                           |

### 15.8.3 이미지 최적화 [공통, 네이버는 빠른 사이트 선호]

- 형식 우선순위: AVIF > WebP > JPEG (브라우저 지원에 따라 자동 분기)
- responsive sizes: 320px·640px·1024px·1920px
- next/image 사용 시: Hero 첫 이미지는 `priority={true}` + `fetchPriority="high"`
- 작품 페이지 메인 이미지는 server-side render (lazy-load 금지)
- 두 번째 이미지부터 lazy-load

### 15.8.4 회피 항목

- alt 텍스트에 키워드 채우기 ("그림 작품 미술 한국 작가 오윤 칼노래 사기 구매")
- 빈 alt(`alt=""`) — 장식 이미지가 아닌 한 회피
- 같은 alt를 여러 이미지에 복사
- 이미지에 작가명·작품명·가격을 글자로 박아 넣기 — HTML 텍스트로 따로 노출

## 15.9 내부 링크 전략

사이트 내부 링크 구조는 검색엔진이 사이트 위계를 학습하는 가장 강한 신호다.

### 15.9.1 푸터 링크 구조 [공통]

푸터는 모든 페이지에 동일하게 적용되므로 사이트 전체의 내부 링크 척추다. 현행 푸터(탐색·참여·구매·문의)에 다음을 추가.

> **푸터 확장 — 핵심 페이지 직접 링크**
>
> - 작가 16명 직접 링크 (거장 6 + 화제 작가 10) — 운영팀 우선 작가
> - 카테고리 6개 직접 링크 (큐레이터 픽·거장·신진·30만원 이하·인기·시즌)
> - 매거진 카테고리 3개 직접 링크 (작가 이야기·컬렉팅 가이드·미술 산책)
> - 정책 페이지 3개 (개인정보·이용약관·환불 정책)

이렇게 하면 사이트의 모든 페이지가 자동으로 28개의 핵심 페이지를 내부 링크로 받는다.

### 15.9.2 작품 페이지 내부 링크

작품 상세페이지(통합매뉴얼 Part 7의 13섹션 구조) 안에 다음 링크 영역 추가.

- **같은 작가의 다른 작품** (3~6점, 작품 카드 형식)
- **관련 작품** (같은 매체 또는 같은 가격대, 3~6점)
- **이 작가의 매거진 글** (있으면 1~3편)

### 15.9.3 매거진 글 내부 링크

- 본문에서 작품·작가 페이지 자연 링크 (글당 5~15개)
- anchor text는 작품명·작가명을 그대로 (예: "[오윤의 〈칼노래〉](/artworks/{id})")
- 글 하단 "같은 카테고리 다른 글" 3편

### 15.9.4 Anchor text 표준 [공통]

> **Anchor text 작성 규칙**
>
> - Generic 회피: "전체 작품 보기" → "한국 작가 작품 348점 전체 보기"
> - "더 알아보기" → "씨앗페 상호부조 메커니즘 더 알아보기"
> - "About 더 보기" → "씨앗페 협동조합 소개 더 보기"
> - 한국어 자연 어순 (15.1.3)
> - 같은 페이지를 가리키는 anchor text는 어느 정도 일관성 유지하되 100% 동일 회피

## 15.10 매거진 SEO 전략

매거진은 사이트의 가장 큰 informational 검색 자산 잠재력을 가진다. 작품 페이지가 transactional 검색을 잡고, 매거진이 informational 검색을 잡는다.

### 15.10.1 카테고리 3종의 검색 의도 매핑 [공통]

| 매거진 카테고리 | 주된 검색 의도               | 키워드 예시                                                 | 발행 빈도  |
| --------------- | ---------------------------- | ----------------------------------------------------------- | ---------- |
| 작가 이야기     | Informational + Navigational | "오윤 사후판화", "박생광 드로잉 가치", "한국 민중미술 작가" | 매월 1~2편 |
| 컬렉팅 가이드   | Commercial + Informational   | "처음 그림 사기", "그림 진품 구분", "사후판화 진품 보증"    | 매월 1편   |
| 미술 산책       | Informational                | "한국 현대미술 흐름", "민중미술이란", "오방색 한국화"       | 분기 1~2편 |

### 15.10.2 글 제목·H1 설계 — 스마트블록 친화 [공통, 네이버 비중 강함]

네이버에서 작가명·작품명 검색 시 형성되는 세부 주제 블록(스마트블록)을 매거진 글의 제목·H1으로 명시적으로 다룬다.

> **스마트블록 친화 제목 패턴**
>
> - 질문형: "오윤의 사후판화는 진품인가요? — 작가 사후 인쇄본의 진품성"
> - 비교형: "박생광 드로잉이 채색화와 다른 점"
> - 가이드형: "한국 판화 컬렉팅 입문 — 무엇을 봐야 하나"
> - 가격형: "오윤 작품 가격대 — 사후판화부터 드로잉까지"

### 15.10.3 글 본문 표준

- 길이: 정보성 글 1,500자 이상, 작가 인터뷰 800자 이상
- 구조: H2·H3로 단락 분리, 한 단락 3~5문장
- 이미지: 글 본문에 작품 이미지 2~5점, 모두 alt 표준 적용
- 내부 링크: 본문에 5~15개 (작품·작가·다른 글)
- 외부 링크: 신뢰할 만한 외부 자료 1~3개 (위키, 미술관, 학술 자료)

### 15.10.4 저자 표기 — 2026 Authenticity Update 핵심 [Google 중심, 네이버 간접 효과]

> **저자 표기 표준 — E-E-A-T·C-RANK 양쪽 핵심**
>
> - 글 상단에 저자 이름 + 1~2줄 약력 + 프로필 이미지(원형, 60px)
> - Person schema 마크업(15.6.4)
> - 저자별 페이지(/authors/{slug}) 도입은 24개월 검토
> - 익명 글 회피 — 부득이한 경우 "씨앗페 편집팀" 명의

### 15.10.5 운영 부담

- 매월 1~2편 매거진 발행 — 현재 3명 기준 부담 중간
- 매주 1편 발행 — 매출 1500만원/월 추가 후 인력 +1명 시 가능 (총 4명)
- 분기 외부 필자 기고 1편 — 운영 부담 낮음 (협력 매체와 기획)

## 15.11 작가 페이지 SEO

**작가명 검색은 네이버 검색의 비중이 크므로 양쪽 동시 최적화 필수.** 통합매뉴얼 Part 9.5의 작가 16명 우선 + 96명 점진 진화와 정합.

### 15.11.1 작가 단계별 페이지 깊이 [공통]

| 작가 단계 | 작가 수 | 약력 길이  | 대표작 노출 | 매거진 글  | 우선 순위      |
| --------- | ------- | ---------- | ----------- | ---------- | -------------- |
| 거장      | 6명     | 500자 이상 | 5~10점      | 1~3편 필수 | 6개월 안 완성  |
| 화제 작가 | 10명    | 200~400자  | 3~5점       | 1편 권장   | 12개월 안      |
| 일반 작가 | 96명    | 100~200자  | 2~3점       | 선택       | 매출 따라 점진 |

### 15.11.2 작가 페이지 필수 요소

- H1: "{작가명} ({생몰년도})"
- Sub-H1: "{한국 매체 분야} 거장/작가" 같은 검색 카피
- 약력 본문 — 한국어 자연 어순, 시기·매체·주요 활동 명시
- 대표작 ItemList (가격 노출 포함)
- 작가 인터뷰·매거진 글 링크
- Person schema (15.6.3)
- 메타 타이틀·디스크립션 (15.7.1)

### 15.11.3 거장 작가의 매체별 검색 노출 전략 [공통]

거장 작가는 작품명·매체별 long-tail 검색이 강하다. 작가 페이지 안에 매체별 섹션을 둔다.

> **거장 작가 매체별 섹션 예시 — 오윤 페이지**
>
> - 사후판화 섹션 — "오윤 사후판화" 검색 진입로
> - 드로잉 섹션 — "오윤 드로잉" 검색 진입로
> - 대표 작품 섹션 — "오윤 칼노래" 같은 작품명 검색 진입로
> - 약력 섹션 — "오윤 작가" 일반 검색 진입로

각 섹션에 H2·H3 구조 + 매체별 작품 ItemList. 통합매뉴얼 Part 5의 매체별 진품 시그널과 정합.

## 15.13 다국어 — hreflang

현재 사이트의 KO/EN 분기에 대한 hreflang 표준.

### 15.13.1 hreflang 태그 표준 [Google 중심]

각 페이지 `<head>`에 다음을 둔다.

```html
<link rel="alternate" hreflang="ko" href="https://www.saf2026.com/{path}" />
<link rel="alternate" hreflang="en" href="https://www.saf2026.com/en/{path}" />
<link rel="alternate" hreflang="x-default" href="https://www.saf2026.com/{path}" />
```

### 15.13.3 네이버 영문 사이트 노출

네이버는 한국어 콘텐츠가 주 대상이므로 영문 사이트의 네이버 노출은 부수 효과. 별도 최적화는 본 매뉴얼 범위 밖.

## 15.14 기술 SEO — Core Web Vitals·모바일·크롤

### 15.14.1 Core Web Vitals 기준 [Google 중심, 네이버도 빠른 사이트 선호]

| 지표                            | 목표       | 실패 시 영향                       |
| ------------------------------- | ---------- | ---------------------------------- |
| LCP (Largest Contentful Paint)  | 2.5초 이하 | Hero 이미지 지연 시 검색 순위 영향 |
| CLS (Cumulative Layout Shift)   | 0.1 이하   | 페이지 로딩 중 레이아웃 흔들림     |
| INP (Interaction to Next Paint) | 200ms 이하 | 클릭·터치 반응 지연                |

### 15.14.2 Hero 슬라이드의 LCP 처리

통합매뉴얼 Part 6.4의 Hero 옵션 C는 큰 단일 작품 이미지 + 5~6 슬라이드 구조. 검색 측면에서 다음 적용.

> **Hero 슬라이드 LCP 보호**
>
> - 첫 슬라이드 이미지는 server-side render
> - `<img priority fetchpriority="high">` 또는 next/image의 `priority={true}`
> - 나머지 슬라이드는 lazy-load
> - 슬라이드의 작품명·작가명·가격은 HTML 텍스트로 노출 (이미지 박힘 회피)

### 15.14.3 모바일 퍼스트 인덱스 [공통]

통합매뉴얼 0.8의 모바일 우선 설계와 자연 정합. 추가 점검 항목.

- 모바일 viewport 메타 (이미 적용)
- 모바일 폰트 크기 14px 이상
- 터치 영역 44px 이상 (이미 적용)
- 모바일·데스크탑 같은 콘텐츠 (cloaking 회피)

### 15.14.4 robots.txt 표준 [공통, 네이버 Yeti 명시]

```
User-agent: *
Allow: /
Disallow: /admin/
Disallow: /api/
Disallow: /*?sort=*
Disallow: /*?filter=*

User-agent: Googlebot
Allow: /

User-agent: Yeti
Allow: /

Sitemap: https://www.saf2026.com/sitemap.xml
```

### 15.14.5 사이트맵 구조 [공통, 네이버 단일 사이트맵 제약]

> **sitemap index 구조로 양쪽 충족**
>
> - 네이버는 사이트당 1개 사이트맵만 받음
> - Google은 복수 가능
> - 해결: `sitemap.xml`을 sitemap index로 만들어 그 안에 분할 sitemap 참조
> - 양쪽에 동일 URL(`/sitemap.xml`) 제출

`sitemap.xml` (sitemap index):

```xml
<?xml version="1.0" encoding="UTF-8"?>
<sitemapindex xmlns="http://www.sitemaps.org/schemas/sitemap/0.9">
  <sitemap><loc>https://www.saf2026.com/sitemaps/artworks.xml</loc></sitemap>
  <sitemap><loc>https://www.saf2026.com/sitemaps/artists.xml</loc></sitemap>
  <sitemap><loc>https://www.saf2026.com/sitemaps/stories.xml</loc></sitemap>
  <sitemap><loc>https://www.saf2026.com/sitemaps/collections.xml</loc></sitemap>
  <sitemap><loc>https://www.saf2026.com/sitemaps/pages.xml</loc></sitemap>
</sitemapindex>
```

### 15.14.6 서버·크롤러 통과

- HTTPS 필수 (이미 적용)
- 서버 방화벽에서 네이버 크롤러(Yeti) IP 허용 확인
- JavaScript 렌더링 — Hero 슬라이드의 후속 슬라이드가 lazy-load일 경우 Googlebot이 못 볼 수 있음. 첫 슬라이드는 SSR 필수.
- canonical 태그 — 모든 페이지에 self-canonical 적용 (필터·정렬 페이지는 메인 카테고리 URL을 canonical로)

## 15.16 네이버 알고리즘 대응 [네이버 전용]

외부 웹사이트가 네이버 알고리즘에 대응할 수 있는 범위. 네이버 알고리즘은 자사 채널(블로그·카페)에 주로 적용되지만, 외부 사이트에도 다음 신호로 작용한다.

- **단일 분야 일관성** — 통합매뉴얼의 페르소나 B 단일화와 매거진 카테고리 3종(미술 전문) 고정이 자연 정합
- **저자 표기** (15.10.4) — Person schema의 jobTitle, url
- **재방문·체류 시간** — 통합매뉴얼 Part 6.7 재방문자 차별과 Part 10 8단계 퍼널과 정합

### 15.16.2 DIA+ 대응 — 검색 의도 정교화

DIA+는 가격·최신성·구체 정보를 선호한다.

- **가격 정보 노출** — 작품 페이지·카드·메타에 가격 명시 (이미 적용)
- **최신 갱신 정보** — 매월 큐레이션 갱신·매거진 발행이 자연 정합
- **구체 정보** — 작가 생몰년도·매체·크기·연도 등 구체 메타데이터

### 15.16.3 스마트블록 대응 — 매거진 콘텐츠 구조

15.10.2의 스마트블록 친화 제목 패턴. 작가명·작품명 검색 시 형성되는 세부 주제 블록을 매거진 글의 제목·H1으로 명시적으로 다룬다.

### 15.16.4 한국어 NLP 친화 — 자연 어순

15.1.3의 한국어 자연 어순 원칙. 모든 검색 카피에 적용. 네이버의 자연어 처리가 매우 한국어 친화적이므로 이 점이 결정적.

## 15.17 E-E-A-T 강화 — 2026 Authenticity Update 대응 [Google 중심, 네이버 간접 효과]

## 15.18 외부 신호 — 백링크·언론보도 [공통, 네이버 자체 채널은 별도]

## 15.20 검색 데이터의 본질 영역 처리 규정 [공통]

통합매뉴얼 14.1의 "본질 영역 자동 변경 금지" 원칙이 검색 데이터에 어떻게 적용되는지를 명시한다.

### 15.20.1 데이터 기간별 처리

> **검색 데이터 기간별 처리 규정**
>
> - **단기 (1~3개월)** — 기록만, 변경 금지. 일시적 변동일 가능성.
> - **중기 (6개월)** — 운영 영역 부분 개선 가능. 노출 영역(메타·카테고리 설명·매거진 제목)의 카피 다듬기.
> - **장기 (12개월+)** — 본질 영역 검토 시작. 단 자동 변경 금지, 이사장님 결정 영역.
> - **매우 장기 (24개월+)** — 매뉴얼 큰 갱신 검토.

### 15.20.2 A/B 테스트와 본질 영역

- A/B 테스트로 본질 카피(Hero·자긍심 박스)를 자동 변경 금지
- 노출 영역의 카피(메타·카테고리 설명)는 운영팀 자율 A/B 가능
- A/B 결과가 본질 변경 권유로 이어질 경우 통합매뉴얼 14.1 충돌 보고 시스템

### 15.20.3 Google·네이버 데이터 충돌 시

- 한쪽 노출 증가, 다른 쪽 감소가 동시 발생할 경우 → 통합매뉴얼 14.1 보고
- 이사장님 결정 전까지 변경 보류
- 페르소나 B 정합성을 깨는 데이터 변동은 검색 데이터라 해도 본질 변경 근거 아님

### 15.20.4 검색 데이터의 본질

> **검색 데이터의 정의**
>
> - 검색 데이터는 사용자의 어휘 평균값이지 본질의 진단이 아니다
> - "사람들이 '저렴한 그림'으로 검색한다" → 노출 영역에 "저렴한"이 들어갈 수 있는 자리(메타·카테고리 설명)에 자연 반영
> - 그러나 본질 영역(자긍심 박스의 "안목 있는 컬렉터")은 변경 근거 아님
> - 본질 영역은 페르소나 B의 누적 신뢰 형성용이며 검색어 평균과 정합할 필요 없음

## 15.21 절대 회피 [공통, 네이버 게시물 운영정책 별도]

매뉴얼이 명시적으로 피해야 할 것.

> **절대 회피 — Google·네이버 공통**
>
> - 키워드 채우기 (메타·alt·본문)
> - 도어웨이 페이지 (검색 진입 후 다른 페이지로 강제 이동)
> - 클로킹 (Googlebot에게 다른 콘텐츠 노출)
> - 숨겨진 텍스트·링크 (시각적 비노출 키워드)
> - 구매 백링크·링크 스팸
> - 기계 생성 콘텐츠 검수 없이 대량 발행 (AI 초안은 검수 필수)
> - 작품 가격·진품성에 대한 과장
> - 페르소나 B 격조 손상하는 클릭베이트
> - 중복 콘텐츠 (같은 작품 여러 URL)

> **절대 회피 — 네이버 게시물 운영정책 추가**
>
> - 도배성 페이지 (유사 구조 페이지 대량 발행)
> - 동일·유사 콘텐츠 대량 발행
> - 어뷰징성 백링크 (자체 사이트 다수 운영해 상호 링크)
> - 네이버는 이 영역을 Google보다 엄격하게 처벌

## 15.24 본질 영역 충돌 보고 권장 사항

검색 매뉴얼이 통합매뉴얼의 본질 영역에 대해 발견한 점. **결정 형태가 아닌 충돌 보고**로 둠. 이사장님 결정 영역.

### 15.24.1 메인페이지 H1 위치

- 통합매뉴얼 Part 6.2의 Hero 카피("내 맘에 드는 그림을 사고...")가 본질 카피로 보호됨

- 권장: Hero 카피는 그대로, H1은 About 영역 또는 시각적으로 작은 보조 위치에 검색 카피로 분리
- 이사장님 결정 사항

### 15.24.2 About 정체성의 키워드 보강

- 통합매뉴얼 Part 6.4의 About 톤 원칙("미술 플랫폼 우선, 상호부조 보조")은 정확
- 검색 측면에서 "한국 작가 112명, 작품 365점"의 "112명·365점"은 검색되지 않는 표현
- 권장: About 영역에 보조 문구로 "한국 현대미술 작가 112명의 회화·판화·사진·조각 원본 직매" 자연 노출
- 본질 카피의 위계는 그대로 유지하되 검색 카피가 자연스럽게 공존하는 형태
- 이사장님 결정 사항

### 15.24.3 카피의 "직매" 표현

- 사이트 메타·일부 영역에 "직매" 사용 — 검색 노출 효과 큼
- 통합매뉴얼의 본질 카피에는 "직매"가 직접 등장하지 않음
- 권장: 검색 카피 영역(메타·카테고리 설명·anchor text)에 "직매"를 자연 노출하되, 본질 카피(Hero·자긍심 박스)는 그대로
