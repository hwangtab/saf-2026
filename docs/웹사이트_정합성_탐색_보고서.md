## ⚠️ 잠재적 개선 사항

### 1. 사용하지 않는 i18n namespace (dead keys)

아래 top-level keys가 코드에서 `useTranslations()`로 호출되지 않습니다. 기능적 오류는 아니나, 유지보수 비용(번역 파일 용량 증가, 혼란)의 원인이 됩니다.

| Namespace              | 설명                                                                                |
| ---------------------- | ----------------------------------------------------------------------------------- |
| `home`                 | `pageErrors.home`은 사용되나, top-level `home` 키 자체는 미사용                     |
| `statistics`           | 홈 통계 데이터는 `lib/site-stats.ts`에서 직접 상수로 참조, i18n 메시지 미사용       |
| `seo`                  | SEO 메타는 코드에서 동적 생성, JSON 파일의 `seo` 키 미사용                          |
| `artworksPage`         | 작품 페이지 타이틀/설명 등이 여기에 있으나 코드에서 직접 문자열 사용 또는 동적 생성 |
| `artistPage`           | 작가 페이지 관련 i18n 키 — 미사용                                                   |
| `relatedArticles`      | 관련 기사 섹션 — 코드의 하드코딩 카피로 대체됨                                      |
| `language`             | 언어 표시 — `LanguageSwitcher`에서 직접 영어/한국어 문자열 사용                     |
| `notFound`             | 404 페이지 — 에러 페이지에서 `pageErrors.home` 사용, `notFound` 미호출              |
| `categoryPage`         | 카테고리 페이지 — `getCategoryDisplayName()`으로 동적 라벨 생성                     |
| `petition` (top-level) | `petition.ohYoon`은 사용되나 top-level `petition`의 다른 키 미사용 가능성           |

**권장**: 사용하지 않는 namespace에 대한 실제 사용 여부 확인 후 정리. 현재는 기능에 영향 없음.

---

### 2. `/archive` 페이지 — 하위 라우트와 중첩 구조

`/archive` 페이지(`app/[locale]/archive/page.tsx`)가 존재하며, `/archive/2026` 및 `/archive/2023`도 각각 별도 페이지로 존재합니다. Next.js App Router에서 동일한 prefix의 페이지와 하위 라우트가 공존하는 것은 의도된 구조이나, SEO 관점에서 duplicate content 위험이 있습니다.

- **현재**: `sitemap.ts`에서 `/archive`와 `/archive/2026`, `/archive/2023`가 모두 entry로 발행됨
- **권장**: `/archive`를 인덱스 페이지로 유지하되, `/archive/2026` 등의 상세 페이지에 canonical 설정으로 통합 확인

---

### 3. EXHIBITION.DATE — 종료된 이벤트의 날짜 표시

```typescript
// lib/constants.ts
DATE: '2026년 1월 14일 - 2026년 1월 26일',
```

오프라인 전시는 2026년 1월 26일에 종료되었으나, `EXHIBITION.DATE` 상수는 원본 날짜를 유지합니다. `/archive/2026` 페이지에서 "본 오프라인 전시는 2026년 1월 26일에 종료되었습니다" 안내로 우회 중이나, 상수 자체의 시맨틱이 과거 이벤트를 가리킵니다.

- **현재**: 페이지에서 "종료됨" 텍스트로 보완
- **권장**: `EXHIBITION.PAST_DATE` 또는 `EXHIBITION.HISTORICAL_DATE`로 분리하여 현재 시점과 명확히 구분
