# 거장 컴포넌트 4명 본문 심화 설계

## 목표

컴포넌트 4명(신학철·민정기·이철수·박불똥)의 본문을 **박생광 페이지 수준의 깊이**로 끌어올려
6명 거장 페이지의 콘텐츠 격차를 해소한다.

## 비목표 (YAGNI)

- special 페이지 2명(박생광·오윤) 보강 안 함 — 이미 풍부, ROI 낮음.
- 새 라우트·새 컴포넌트 파일 생성 안 함 — 기존 4개 파일에 섹션 추가/확장.
- portrait 사진 추가는 **이미지 자산이 있을 때만** (없으면 생략, 본 작업에서 새 이미지 제작 안 함).
- 레이아웃/디자인 시스템 변경 안 함 — 기존 시각 모티프·브랜드 토큰 유지.
- 6명 공통 섹션 구조 통일 안 함 (접근 B 기각) — 작가별 고유 서사 유지.

## 접근: 작가별 고유 심층화 (접근 A)

박생광 페이지 패턴을 4명에 이식하되, 각 작가의 핵심 주제로 **고유 서사**를 구성한다.
표준 템플릿 평준화가 아니라, 박생광처럼 작가마다 다른 named 심층 섹션을 둔다.

### 각 페이지 공통 보강 패턴

1. **bio 내러티브 3문단 → 5문단+ 심화** — 기존 서사를 더 구체적 사실·맥락으로 확장
2. **핵심 주제 sub-essay 섹션 신설** — 작가별 2~4개 named h3 심층 에세이
   (박생광의 "파괴된 터전에서 / 30년의 혐의 / …" h3 다수 구조 차용)
3. **작가 quote 추가/보강** — 출처 검증된 실제 발언만 (박불똥·신학철·민정기 신규)
4. **전시·소장 카드 보강** — 구체 전시명·연도·소장처. 신학철 "개인전 다수" → 구체화
5. **연표 구체화** — 이철수·박불똥의 "1990s~/현재" 모호 항목을 구체 연도·사건으로

### 작가별 잠정 심층 주제 (구현 시 WebSearch로 확정)

각 작가당 검증 후 2~4개 sub-essay 채택. 아래는 방향성이며, 검증 안 되는 주제는 제외한다.

- **신학철**: ① 「모내기」 사건 전모(1987 제작 → 1989 압수·구속 → 1·2심 무죄 → 1999 대법원
  유죄 → 2004 UN 자유권규약위 → 2018 MMCA 위탁) 심층 박스 ② 「한국근대사」·「한국현대사」
  연작의 수직 몽타주 형식 분석 ③ 「비상탈출」 등 1970년대 초기작
- **민정기**: ① 현실과 발언 창립(1980)과 1980년대 도시·민중 연작 ② 2018 판문점 「북한산
  전도」 심층(기존 박스 확장) ③ 1987 양평 정착 후 진경산수·풍경으로의 전환
- **이철수**: ① 1980년대 민중판화(「두렁」 등) ② 1987 충북 제천 낙향과 선(禪)·일상으로의 전환
  ③ 나뭇잎 편지·달력·엽서를 통한 대중적 확산
- **박불똥**: ① 포토몽타주/콜라주 기법의 정치성 ② 1980년대 민족미술협의회·민중미술 운동 참여
  ③ 신군부·자본 비판 대표 연작

## 사실성 원칙 (엄격 — 위반 시 구현 거부)

- **모든 추가 전기·전시·인용은 WebSearch 교차검증.** 1차 출처/권위 매체 우선, 가능하면
  hyperlink (`target="_blank" rel="noopener noreferrer"`). 기존 `content/artist-articles.ts`의
  큐레이션 링크 재활용 가능.
- **작가 quote는 출처 있는 실제 발언만.** 위조·각색·"있을 법한" 인용 절대 금지.
  검증 안 되면 quote 섹션 자체를 넣지 않는다 (빈약해도 거짓보다 낫다).
- 검증 안 되는 sub-essay 주제는 빼고 확실한 주제로만 구성. 작가당 sub-essay 수는 가변
  (박생광 5개가 상한 목표일 뿐, 사실이 받쳐주는 만큼만).
- 캠페인 프레이밍 유지: 거장은 "동료 예술인을 위한 연대자". 금융 피해 당사자 묘사 금지.

## 작업 단위 분해 (independent units)

작가 1명 = 1 독립 task. 4개 task. 각 task 내부 흐름:

1. WebSearch로 해당 작가 사실 검증 (전기·전시·수상·인용)
2. 검증된 사실로 bio 심화 + sub-essay 2~4개 + quote(있으면) + 전시카드 + 연표 작성 (ko/en)
3. type-check + 해당 파일 lint + 색상 규칙 grep
4. 커밋

작가 간 의존성 없음 — 순서 무관, 병렬 가능. 단 각 task는 동일 컴포넌트 패턴을 따르므로
박생광 `app/[locale]/special/park-saenggwang/page.tsx`를 sub-essay 구조 참조 모델로 삼는다.

## 리스크

- **사실 검증 비용**: 4명 × 다수 sub-essay = 상당한 WebSearch. 생존 작가(이철수·민정기·박불똥)는
  최근 인터뷰·전시 정보가 검증 가능하나, quote 출처 확보가 어려울 수 있음 → 그 경우 quote 생략.
- **파일 비대화**: 컴포넌트가 박생광 page.tsx(1131줄) 수준으로 커짐. 단 sibling이 이미 그 크기라
  패턴 일관성 측면에서 허용. 과도하면 작가별 콘텐츠를 별도 데이터 모듈로 분리 검토(구현 시 판단).
