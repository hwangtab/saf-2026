# GA4 + GSC 설정 가이드 — SEO 성과 측정 체계 구축

> 목표: 유기 검색 트래픽 → 작품 구매 전환까지의 퍼널 추적

---

## Part 1: Google Search Console (GSC) 설정

### Step 1: 속성 추가 및 소유권 확인

1. [search.google.com/search-console](https://search.google.com/search-console) 접속
2. **URL 접두어** 방식으로 `https://www.saf2026.com` 추가
3. 소유권 확인 방법 (택 1):
   - **HTML 메타태그** (이미 준비됨): `.env.local`에 `NEXT_PUBLIC_GOOGLE_VERIFICATION` 값 설정
     → `app/layout.tsx`의 `metadata.verification.google`에서 자동 렌더링됨
   - **DNS TXT 레코드**: 도메인 DNS에 google-site-verification TXT 추가

### Step 2: 사이트맵 제출

1. GSC > 사이트맵 > URL 입력: `https://www.saf2026.com/sitemap.xml`
2. 제출 후 상태가 "성공"이 되면 완료
3. 사이트맵은 `app/sitemap.ts`에서 동적 생성 — 현재 모든 공개 페이지 포함

### Step 3: 초기 검수 (24~48시간 후)

| 확인 항목          | GSC 위치                  | 기대값      |
| ------------------ | ------------------------- | ----------- |
| 색인 상태          | 페이지 > 색인 생성        | 200+ 페이지 |
| 사이트맵 상태      | 사이트맵                  | "성공"      |
| 모바일 사용 편의성 | 환경 > 모바일 사용 편의성 | 오류 0      |
| Core Web Vitals    | 환경 > Core Web Vitals    | 양호        |

### Step 4: 키워드 모니터링 설정

**핵심 모니터링 키워드** (GSC > 실적 > 검색어 필터):

```
필터 추가 → "검색어 포함" → 아래 키워드 각각 확인:
- 미술 작품 구매
- 그림 구매
- 현대미술
- 온라인 갤러리
- 컬렉팅
- 씨앗페
- 전시회
```

**주간 체크 루틴**:

1. 실적 보고서 > 최근 7일 vs 이전 7일 비교
2. 노출수 증가 추세 확인
3. CTR 3% 미만 키워드 → 제목/description 개선 대상
4. 순위 변동 큰 페이지 확인

**월간 데이터 백업**: GSC 데이터는 16개월만 보관되므로, 매월 1회 실적 보고서 CSV 다운로드 → Google Drive 보관

## Part 2: GA4 설정

### Step 1: GA4 속성 생성 및 연동

1. [analytics.google.com](https://analytics.google.com) 접속
2. 속성 만들기 > 웹 > `https://www.saf2026.com`
3. 측정 ID (G-XXXXXXX) 복사
4. `.env.local`에 추가:
   ```
   NEXT_PUBLIC_GA_ID=G-XXXXXXX
   ```
5. 이미 `@vercel/analytics`와 `@vercel/speed-insights`가 설치되어 있어 기본 페이지뷰/Web Vitals는 Vercel 대시보드에서도 확인 가능

### Step 2: GA4 ↔ GSC 연동

1. GA4 > 관리 > 속성 > Search Console 링크
2. 같은 Google 계정으로 연결
3. 24시간 후 데이터 수집 시작
4. 연동 후 GA4에 "Search Console" 보고서 카드가 추가됨

### Step 3: 전환 이벤트 설정

| 이벤트명            | 트리거                | 용도            |
| ------------------- | --------------------- | --------------- |
| `purchase_complete` | 결제 성공 페이지 도달 | 노스스타 메트릭 |
| `add_to_cart`       | 구매 버튼 클릭        | 퍼널 중간 단계  |
| `artwork_view`      | 작품 상세 페이지 조회 | 관심도          |
| `magazine_read`     | 매거진 글 조회        | 콘텐츠 성과     |
| `share_click`       | 공유 버튼 클릭        | 바이럴 지표     |

> 현재 `@vercel/analytics`의 `track()` 함수가 `TrackingLink.tsx`에서 사용 중.
> GA4 이벤트를 별도로 설정하려면 `gtag('event', ...)` 호출 필요.

### Step 4: SEO 대시보드 구성

#### 방법 A: GA4 기본 보고서 활용

1. **유기 검색 트래픽**: 획득 > 트래픽 획득 > "Organic Search" 필터
2. **랜딩 페이지 성과**: 참여도 > 랜딩 페이지 > "Organic Search" 세그먼트
3. **전환 경로**: 전환 > 전환 경로 > 첫 번째 터치포인트 "Organic Search"

#### 방법 B: Looker Studio 커스텀 대시보드 (권장)

1. [lookerstudio.google.com](https://lookerstudio.google.com) 접속
2. 데이터 소스 추가: GA4 + Search Console
3. 아래 차트 구성:

**대시보드 레이아웃:**

