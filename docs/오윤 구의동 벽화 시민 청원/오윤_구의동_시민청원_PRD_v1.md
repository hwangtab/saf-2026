## 0. 한 줄 요약 (TL;DR)

오윤(1946–1986)이 1974년 서울 광진구 구의동에 양면으로 새긴 테라코타가 건물 매각·8월 철거로 멸실 위기에 처했다. 매수인·우리은행·유족이 만든 보존의 흐름에 시민의 이름을 더해, **차기 서울시장에게 작품의 안전한 해체·보존·이관을 해결해 주시기를 청원**한다. SAF2026 사이트 안에 단일 랜딩 페이지를 두고 **자체 서명 폼(Supabase 백엔드)**을 운영하며, **2026-05-10까지 1만 명의 서명**을 실시간 카운터와 함께 모은다.

## 1. 배경 (Context)

### 1.1. 작품 및 작가

- **오윤 (1946–1986)**: 한국 현대미술의 대표 작가. 옥관문화훈장 추서(2005), 국립현대미술관 회고전(2006). 작품 약 400여 종.
- **대상 작품**: 1974년, 28세의 오윤이 서울 광진구 구의동의 상업은행(현 우리은행 계열) 건물에 **양면으로 새긴 테라코타 벽화**. 청년 작가가 공공미술 작품을 의뢰받기 어려웠던 시대의 흔치 않은 한국 공공미술 초기 작품.

> **콘텐츠 표기 원칙 (v1.1 반영)**: 페이지 본문에서는 "벽" 단독 표현을 피하고 **"테라코타" / "작품"**으로 통일한다. 벽화가 회화적 평면이라는 인상을 주지 않도록 한다. **양면 새김**이라는 사실은 작품 이야기 섹션과 보도자료에서 보존 가치의 핵심 근거로 적극 활용한다.

### 1.2. 위기 상황

- 2026년 봄 해당 건물이 매매됨.
- **건물 8월 철거 후 신축 예정.** 그 과정에서 작품이 멸실될 위험.
- **작품 해체·이관 마지노선: 2026년 8월 초.**
- 추진단 내부 협의된 보존 해체 작업 윈도우: **2026-07-21 ~ 08-15.** 페이지 노출 표현은 "8월 초까지"로 통일(v1.1).

### 1.3. 이미 만들어진 흐름 (캠페인의 토대)

이 청원은 무에서 시작된 운동이 아니다. 다음 세 흐름이 이미 만들어져 있고, 시민 서명은 그 위에 올려지는 마지막 한 발이다.

1. **우리은행** — 매매 과정에서 매수인에게 작품의 존재를 알려줌.
2. **매수인** — 작품 가치를 알아보고 국립현대미술관 문의, 유족 직접 연락.
3. **유족** — 보존과 공공 이관을 적극적으로 원함.
4. **한국스마트협동조합** — 유족과의 협의로 보존 해체·공공 이관 추진을 맡고, **매수인과 작품 반출 동의서를 체결**.

### 1.4. 시점의 무게

| 시점                     | 의미                                                                            |
| ------------------------ | ------------------------------------------------------------------------------- |
| 2026년                   | 오윤 서거 **40주기**                                                            |
| 2026-06-03               | **제9회 전국동시지방선거** — 차기 서울시장 결정                                 |
| 2026-05-10               | 본 청원 마감 (후보자 등록 전 시민의 의지를 모아 전달)                           |
| 2026년 8월 초 (마지노선) | 작품 해체·이관 마감 — 건물 철거 전 분리 필요. 추진단 내부 작업 윈도우 7-21~8-15 |

### 1.5. 씨앗페(SAF2026) 캠페인과의 관계

씨앗페의 본 캠페인은 **한국 예술인 금융 차별 해소를 위한 상호부조**다. 본 청원은 결이 다른 사안(공공미술 보존)이지만, 다음 두 축에서 연결된다.

- **운영 주체 동일**: 두 캠페인 모두 한국스마트협동조합이 추진. 씨앗페가 축적한 시민·미술계 네트워크가 청원 확산의 토대가 된다.
- **공통 가치**: 한국 예술인의 사회적 지위와 작품의 공공적 가치를 회복하려는 동일한 문제의식. 살아 있는 예술인의 금융 환경(씨앗페)과, 떠난 예술인의 작품(오윤 청원)을 함께 지킨다.

> **콘텐츠 처리 원칙**: 페이지 안에 **"씨앗페가 이 청원을 함께하는 이유"** 한 단락(120~180자)을 두어, 사용자가 두 캠페인을 헷갈리지 않도록 명시한다. 출품 작가들을 "금융 피해 당사자"로 프레이밍하지 않는 씨앗페 원칙(`CLAUDE.md`)을 본 페이지에서도 동일하게 유지한다.

## 2. 목표 및 성공 지표

### 2.1. Primary Goal

**2026-05-10까지 1만 명의 서명을 모아, 서울시장 후보 측에 청원을 전달한다.**

### 2.2. Secondary Goals

- 「오윤 작품 보존 추진위원회」 발족 선언문에 오를 **시민 추진위원 200명** 확보.
- 차기 서울시장에게 전달할 **시민 메시지** 풀 확보 (공개 동의 기준 100건 이상).
- 청원 캠페인을 통해 **씨앗페 인지 증가**: 페이지 방문자의 N% 이상이 씨앗페 메인으로 이동.

### 2.3. KPI (정량 지표)

| 지표                    | 목표               | 측정 방식                              |
| ----------------------- | ------------------ | -------------------------------------- |
| 총 서명자 수            | 10,000             | Google Form 응답 수                    |
| 거주 지역 분포(서울)    | 25개 구 모두 ≥1명  | Form 응답 집계                         |
| 시민 추진위원 신청자 수 | ≥200               | Form 4번 항목                          |
| 청원 메시지(공개 동의)  | ≥100               | Form 5번 항목 + 별도 공개 동의         |
| 페이지 → 폼 전환율      | ≥15%               | GA: petition_view → petition_form_open |
| 폼 시작 → 제출 완료율   | ≥70%               | Form analytics                         |
| 페이지 평균 체류 시간   | ≥45초              | GA                                     |
| 모바일 비율             | 추적용 (예상 ≥80%) | GA                                     |

### 2.4. 정성 목표

- 시민이 **"내가 짊어지는 부담"이 아니라 "이미 진행되는 일에 손을 더한다"**는 감각으로 서명한다.
- 작품 사진 없는 출시는 하지 않는다. 페이지의 신뢰가 출시 시점부터 무겁게 선다.

## 4. 범위 (Scope)

### 4.1. In Scope (v1, 출시본)

- SAF2026 도메인 내 **단일 랜딩 페이지** 1종 (`/petition/oh-yoon`).
- **자체 서명 폼 + Supabase 백엔드** — 페이지 안에서 인라인 서명. 외부 redirect 없음. (§10.2 / §10.4)
- **이메일 OTP 인증** — 봇·중복 방지 + 실명성 확보. Supabase Auth `signInWithOtp` 활용.
- **실시간 카운터** — Supabase 집계 SQL/Edge Function. SSR 또는 클라이언트 폴링으로 초~분 단위 갱신.
- 카카오톡·페이스북·트위터·문자(Web Share API)·링크 복사 공유 버튼.
- 사전 작성된 **공유 메시지 템플릿 3종** (카톡 / 문자 / SNS).
- 진행 상황 안내 메일 자동 발송 (서명 직후 영수증 + 마감 직전 안내 + 결과 통지).
- 운영자 최소 화면 — `/admin/petition/oh-yoon`(`requireAdmin`). 실시간 집계·부적절 메시지 마스킹·CSV 내보내기.
- 「자세히 보기」 PDF 제공 — 시민제안서 5판.
- Footer · 개인정보 처리방침 링크.

### 4.2. Out of Scope (v1)

- **Google Form 사용 안 함** (의뢰자 결정, v1.2). 폼 콘텐츠 명세서는 **필드·문구·동의문 레퍼런스**로만 사용.
- 결제·후원금 수령 — **이번 청원에서는 후원을 받지 않는다.**
- 다국어 풀 번역 — 영문 locale은 한국어 페이지로 폴백 또는 영문 단신 안내 페이지.
- 추진위원 전용 포털·게시판.
- SMS 알림 (이메일만 v1).

### 4.3. Future (v2 이후)

- 영문 안내 페이지(해외 한국미술 연구자 대상).
- 후속 캠페인 페이지(보존 이관 진행 상황 기록) — 청원 종료 후 별도 라우트로.
- 추진위원 전용 미니 게시판·정기 뉴스레터.
- 지역별·세대별 서명 분포 인포그래픽 자동 생성.

## 5. 사용자 여정 (User Journey)

### 5.1. Happy Path

```
[1] 외부 진입
    카카오톡·SNS·뉴스·이메일 → /petition/oh-yoon
        ↓
[2] HERO (5~10초)
    "멸실될 위기의 오윤 벽화, 우리가 구하자."
    + 마감 D-N + 진행률 바 + [지금 서명하기 →]
        ↓
[3] 위기 한 줄 → 작품 이야기 → 세 가지 일 (이미 진행되는 흐름)
        ↓ (공감 + 신뢰 형성)
[4] 청원 한 줄 강조 박스 (공유 가능)
        ↓
[5] 참여 방법 3단 (서명 / 추진위원 / 공유)
        ↓
[6] 시점의 무게 → 추진 주체 → FAQ → 결구
        ↓
[7] [지금 서명하기 →] 클릭 → Google Form (새 탭)
        ↓
[8] 폼 작성 (30초)
    동의문 → 성함 → 이메일 → 거주 지역 → 추진위원 여부 → 메시지(선택)
    → 개인정보 동의 → 제출
        ↓
[9] 감사 메시지 (폼) + 공유 유도
        ↓
[10] 진행 상황 안내 메일 (운영팀 발송)
```

### 5.2. 주요 진입 채널 (예상)

- 카카오톡 1:1 / 단톡방 공유 (가장 큰 비중)
- 페이스북 / 트위터 / 인스타그램
- 이메일 (씨앗페 뉴스레터 구독자, 미술계 네트워크)
- 언론 기사 본문 링크
- 직접 입력 (오프라인 행사·홍보물에 노출된 짧은 링크)

## 7. 비기능 요구사항 (Non-Functional)

### 7.1. 성능

- **LCP < 2.5s** (모바일 4G 가정) — HERO 이미지 priority + WebP/AVIF.
- **CLS < 0.1** — 진행률 바·카운터 placeholder 고정 높이.
- **TTFB < 600ms** — Vercel Edge / 정적 출력.
- 이미지 최적화: `next-image-export-optimizer`(로컬) / `SafeImage`(원격) 사용. **`next/image` 직접 import 금지** (`CLAUDE.md`).

### 7.2. 접근성 (WCAG 2.1 AA)

- 색 대비 ≥4.5:1 — 본문 텍스트는 `text-charcoal-deep` / `text-charcoal` / `text-gray-800` 토큰, 보조는 `-muted` / `gray-600` 사용.
- 키보드 네비게이션 — 모든 CTA·링크·아코디언 Tab 접근 가능, focus ring 노출.
- alt 텍스트 — 작품 사진은 작가/연도/매체 포함 ("오윤, 1974, 테라코타 벽화, 서울 광진구 구의동 (사진: 출처)").
- 아코디언은 `aria-expanded`, `aria-controls` 적용.
- 진행률 바는 `role="progressbar"` + `aria-valuenow / aria-valuemax`.
- 폰트 크기: 본문 16px 이상, 모바일에서 줌 가능.

### 7.3. SEO 및 메타

- `<title>`: "오윤의 작품을 시민의 품으로 — 차기 서울시장께 드리는 시민 청원 | SAF2026"
- `<meta name="description">`: 위기 한 줄 + 마감일.
- OG 이미지: 1200×630, **벽화 디테일 사진 + 청원 한 줄**. Satori 렌더 시 `BRAND_COLORS` 참조.
- JSON-LD: `Article` + `Organization` 구조화 데이터.
- 동적 sitemap.xml에 등록 (이미지 URL의 query string 제거 — 최근 커밋 `35b0d0a3` 반영).

### 7.4. 보안 / 프라이버시 (자체 폼 운영 기준)

- HTTPS only (Vercel 기본).
- 외부 링크 모두 `target="_blank" rel="noopener noreferrer"` (`CLAUDE.md`).
- **Supabase RLS 정책 필수** — 익명 INSERT 허용(서명 자체), SELECT는 운영자(`role = 'admin'`)만. PII 칼럼은 anonymous 클라이언트에서 절대 SELECT 불가. (§10.4)
- **Service Role Key는 Edge Function 내부에서만 사용**. 클라이언트 번들에 절대 노출 금지.
- **hCaptcha** — 봇 차단. 서버 측 토큰 검증(`https://hcaptcha.com/siteverify`) 통과 시에만 INSERT 진행.
- **중복 차단** — DB unique constraint(`(petition_slug, email_hash)`)로 동일 이메일 재서명 차단. 친절한 에러 응답.
- **Rate limiting** — 동일 IP에서 60초 내 폼 제출 5회 초과 차단 (Vercel Edge Middleware 또는 Server Action 내부).
- 카카오 공유 SDK 키는 환경변수(`NEXT_PUBLIC_KAKAO_JS_KEY`)에서 주입.
- **메시지 필드 XSS 방지** — 저장은 raw, 노출 시 React 기본 escape에 의존. raw HTML 삽입 API 사용 금지.
- **CSP** — 자체 폼이 추가됨에 따라 connect-src에 Supabase URL 명시.

### 7.5. 반응형

- 모바일 우선 (예상 트래픽 80%+ 모바일).
- 브레이크포인트: `sm` 640 / `md` 768 / `lg` 1024 / `xl` 1280 (`CLAUDE.md`).
- HERO는 모바일 한 화면 안에 슬로건·D-N·CTA·진행률 바가 모두 들어가는 압축 레이아웃.
- "세 가지 일"·"참여 방법"·"시점의 무게"의 카드 그리드는 모바일에서 세로 스택 자동 전환.

### 10.4. 데이터 모델 (Supabase Postgres)

```sql
-- 청원 정의(향후 확장 위해 분리)
create table petitions (
  slug          text primary key,         -- 'oh-yoon'
  title         text not null,
  goal          int  not null default 10000,
  deadline_at   timestamptz not null,
  is_active     boolean not null default true,
  created_at    timestamptz not null default now()
);

-- 서명 레코드
create table petition_signatures (
  id              uuid primary key default gen_random_uuid(),
  petition_slug   text not null references petitions(slug),
  full_name       text not null,
  email_hash      text not null,            -- 중복 방지용 (sha256 lowercase email)
  email_encrypted text not null,            -- 메일 발송용 (pgsodium 또는 양방향 암호화)
  region_top      text not null,            -- '서울' / '경기' / '부산' / ... / '해외' (17개 + 해외)
  region_sub      text,                     -- '광진구' / '성남시 분당구' / null (해외·세종 등 단일 행정 단위는 null)
  is_committee    boolean not null default false,
  message         text,                     -- 시장께 한 마디 (선택)
  message_public  boolean not null default false,  -- 공개 동의 여부
  is_masked       boolean not null default false,  -- 운영자 마스킹 처리
  agreed_petition boolean not null,
  agreed_privacy  boolean not null,
  agreed_overseas boolean not null,         -- 국외 이전 동의 (Mumbai 리전 + 메일·hCaptcha)
  user_agent      text,
  ip_hash         text,                     -- sha256(salt + ip), 통계용
  created_at      timestamptz not null default now(),

  unique (petition_slug, email_hash)
);

create index on petition_signatures (petition_slug, created_at desc);
create index on petition_signatures (petition_slug, region_top);

-- 집계 뷰 (RLS로 익명 SELECT 허용)
create view petition_counts as
select
  petition_slug,
  count(*) as total,
  count(*) filter (where is_committee) as committee_total,
  count(distinct region_top) as region_top_count
from petition_signatures
group by petition_slug;
```

### 10.5. RLS 정책

```sql
alter table petition_signatures enable row level security;

-- 익명: INSERT만, 그것도 Edge Function이 service role로 실행 (직접 INSERT 차단)
create policy "no direct anon insert"
  on petition_signatures for insert to anon
  with check (false);

-- 인증된 admin만 SELECT
create policy "admin can read"
  on petition_signatures for select to authenticated
  using (
    exists (select 1 from public.profiles
            where profiles.id = auth.uid() and profiles.role = 'admin')
  );

-- 익명: petition_counts 뷰 SELECT 허용 (집계만 노출, PII 없음)
grant select on petition_counts to anon;
```

### 10.6. 서명 흐름 (hCaptcha + INSERT)

```
[1] 사용자가 폼 작성
        ↓
[2] 폼 하단 hCaptcha widget 통과 (자동 invisible 또는 명시적 클릭 — 접근성 위해 invisible 권장 + fallback)
        ↓
[3] "서명하기" 클릭 → 클라이언트가 Server Action 호출
    payload: { name, email, region_top, region_sub, is_committee, message,
               message_public, agreed_petition, agreed_privacy, hcaptcha_token }
        ↓
[4] Server Action (Node 런타임, service role) 실행:
    - hCaptcha siteverify POST (https://hcaptcha.com/siteverify)
      - 실패 시 422 "봇 의심 — 다시 시도해 주세요"
    - 입력 검증 (이메일 형식, 동의 체크, 지역 코드 유효)
    - email_hash = sha256(PETITION_EMAIL_HASH_SALT + lowercase(email))
    - email_encrypted = pgsodium 또는 양방향 암호화
    - ip_hash = sha256(PETITION_IP_HASH_SALT + ip)
    - INSERT into petition_signatures
      - unique 충돌(`23505`) 시 409 "이미 서명하셨습니다 (영수증 메일을 확인해 주세요)"
    - 영수증 메일 큐잉 (Edge Function 또는 즉시 발송)
        ↓
[5] 성공 응답 → 클라이언트가 카운터 즉시 +1 (낙관적) → 감사 화면 + 공유 5종 버튼
        ↓
[6] 영수증 메일이 사용자 받은편지함에 도착 → 이메일 정확성의 자기검증 효과
    (가짜 이메일이면 메일이 도달하지 않으며, 추진단은 향후 안내 메일에서도 자동 bounce 추적 가능)
```

> **OTP를 쓰지 않는 보강책**: 영수증 메일 발송 → 받은편지함 도착 여부로 이메일 진성 자기검증. bounce 메일은 별도 큐에 자동 마킹하여 통계 제외.

#### 10.7.5. 메시지 모더레이션 워크플로우

```
신규 메시지 INSERT
  ↓
DB 트리거가 메시지를 검토 큐에 자동 등록 (reviewed_at = null)
  ↓
운영자가 [메시지] 탭에서 검토
  - 카드 상단: 작성자 (지역만 노출, 이름·이메일 마스킹) · 작성 시각 · 공개 동의 여부
  - 본문: 메시지
  - 액션: [공개 OK] / [마스킹] / [삭제 (관리자만)] / [추진단에 전달]
  ↓
처리 후 자동
  - reviewed_at = now()
  - reviewed_by = auth.uid()
  - 결정 이력이 petition_audit_log에 기록 (action, before, after)
  ↓
마스킹 처리된 메시지: 청원 결과 발표·전달 자료에서 제외, DB는 보존
삭제 처리된 메시지: 본문만 null로 (감사 로그에 원문 보존, 30일 후 자동 파기)
```

**검토 기준 (운영자 합의)**

- 통과: 청원 취지에 부합, 익명·실명 무관, 정중한 어조
- 마스킹: 욕설·혐오 표현, 다른 시민 신상 노출, 무관한 광고
- 삭제: 명백한 스팸·동일 문구 봇 패턴

**일괄 처리 도구**: 키워드 필터 + 다중 선택 + 일괄 마스킹/통과.

| 추진위 발족식 안내 | 추진위원 한정            | 미리보기 + 테스트 발송 | Stretch                |

UI 요구사항:

- 수신자 수 미리보기 (`SELECT count(*) WHERE …`)
- 본인 메일로 테스트 발송 버튼
- 발송 큐 상태 (대기 / 발송 중 / 완료 / 실패) + 실패 재시도
- 발송 후 bounce·열람률 (Resend webhook)

#### 10.7.7. CSV 내보내기

세 가지 모드 분리:

| 모드       | 포함 필드                              | 권한           | 용도                   |
| ---------- | -------------------------------------- | -------------- | ---------------------- |
| 전체 (PII) | 성함·이메일·거주지·메시지·동의 시각    | admin only     | 후보 측 청원 정식 전달 |
| 마스킹     | 거주지·동의 시각·메시지(공개 동의분만) | admin / 추진단 | 보도자료·통계          |
| 추진위원   | 성함·연락처·서명 시각                  | admin only     | 발족 선언문 명단       |

내보내기 액션도 감사 로그에 기록(누가·언제·어떤 모드·행 수).

#### 10.7.8. 추진위원 관리

- 신청자 표 — 성함·지역·서명 시각·메시지 작성 여부 컬럼.
- 정렬 — 가나다순 (발족 선언문 표준), 서명 시각 역순(최근 신청자 확인).
- 발족 선언문용 텍스트 자동 생성 — 가나다순으로 줄바꿈된 일반 텍스트 (워드 붙여넣기용).
- 추진위원에게만 발송하는 메일 (§10.7.6).

#### 10.7.9. 감사 로그 (`petition_audit_log`)

```sql
create table petition_audit_log (
  id           bigserial primary key,
  actor_id     uuid references auth.users(id),
  action       text not null,            -- 'mask_message' / 'unmask' / 'delete' / 'csv_export' / 'mail_send' / 'sign_force_close' …
  target_type  text,                     -- 'signature' / 'message' / 'campaign'
  target_id    uuid,
  details      jsonb,                    -- before/after, 행 수, 메일 제목 등
  ip_hash      text,
  created_at   timestamptz not null default now()
);

-- RLS: admin만 SELECT, 모두 INSERT 차단(서버에서 service role로만)
```

UI: 감사 로그 탭에서 행 단위 필터(actor·action·기간), 검색, CSV.

#### 10.7.10. 부정행위·이상 신호 탐지

| 신호                    | 조건                                  | 알림              | 자동 조치                               |
| ----------------------- | ------------------------------------- | ----------------- | --------------------------------------- |
| 단일 IP 폭주            | 60s 내 동일 `ip_hash` 5회 시도        | 운영자 메일/Slack | rate-limit 발동(기존 인프라)            |
| 동일 도메인 군집        | 1시간 내 동일 이메일 도메인 50건 초과 | 운영자 알림       | 검토 대기 (자동 차단 X)                 |
| hCaptcha 실패율 급증    | 5분 평균 실패율 > 30%                 | 운영자 알림       | hCaptcha 난이도 높이기 (옵션)           |
| 영수증 메일 bounce 폭증 | 1시간 bounce > 100건                  | 운영자 알림       | 가짜 이메일 군집 의심, 통계 제외 플래그 |
| 카운터 갱신 정지        | 캐시 갱신 5분 이상 정지               | 운영자 알림       | —                                       |

#### 10.7.11. 청원 마감 자동 처리

```
T = PETITION_DEADLINE (2026-05-10 23:59:59 KST)

[T 시점] DB 함수가 자동 실행:
  - petitions.is_active = false 로 전환
  - 신규 INSERT 차단 (RLS / Server Action 검증)
  - 카운터 스냅샷 저장 (petition_snapshot 테이블)

[T 직후] 페이지 자동 변경:
  - 폼 비활성화 → "서명이 마감되었습니다. 함께해 주신 N명께 감사드립니다."
  - 카운터는 동결값 노출

[T + 6개월 (2026-11-10)] cron 자동 파기:
  - PII 컬럼 NULL 처리 (집계 통계는 보존)
  - 감사 로그에 파기 기록
```

cron은 Supabase의 `pg_cron` 또는 Vercel Cron Job으로 운영. v1 출시 시 시스템에 사전 등록.

#### 10.7.12. 권한 등급

- ~~**OQ-1. 거주 지역 처리**~~ → **시·도 + 시·군·구 2단계 드롭다운**. 상세 주소(읍·면·동·번지) 받지 않음. 해외 선택 시 시·군·구 비활성화. (§6.2 FR-FM-09 / §8.2)
