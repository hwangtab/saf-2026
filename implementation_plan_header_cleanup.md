# 헤더 정보구조 정리 실행 계획 (2026-04-09)

## 1) 목적

- 공개 헤더에서 `전시 작품`과 `작품 구매`의 중복 의미를 해소한다.
- 헤더를 `탐색 / 유틸 / 관리자 진입` 구조로 정리해 시각적 혼잡도를 낮춘다.
- 전시/캠페인 사이트 성격에 맞게 메인 헤더를 콘텐츠 탐색 중심으로 재배치한다.

## 2) 현재 확인된 문제

- 데스크톱 헤더 우측에 `검색 / 주문·배송 조회 / 작품 구매 / 관리자 대시보드 / 언어 전환`이 한 줄에 몰려 있다.
- `전시 작품` 메뉴와 `작품 구매` 버튼이 모두 `/artworks`로 연결되어 의미가 중복된다.
- `주문/배송 조회`와 `검색`도 버튼형 스타일이라 `작품 구매`와 같은 레벨로 경쟁한다.
- `관리자 대시보드`는 공개 사이트 헤더에서 시각적 존재감이 커서 운영 도구처럼 보이게 만든다.
- 모바일 메뉴는 데스크톱보다 덜 복잡하지만, 유틸/운영 항목의 위계는 여전히 분리 여지가 있다.

## 3) 목표 상태

- 메인 헤더는 `전시 작품 / 소개 / 매거진 / 아카이브` 중심으로 유지한다.
- `작품 구매` 버튼은 공개 헤더에서 제거한다.
- `검색`은 유지하되 지금보다 가벼운 유틸 스타일로 낮춘다.
- `주문/배송 조회`, `언어 전환`, `관리자 진입`은 메인 탐색보다 약한 보조 요소로 정리한다.
- 모바일에서도 메인 탐색과 보조 유틸이 구분되어 보이도록 맞춘다.

## 4) 구현 범위

### A. 데스크톱 헤더 정리

- `components/common/Header/DesktopNav.tsx`
  - `/artworks`로 연결되는 `작품 구매` CTA 제거
  - `검색` 버튼 스타일 약화
  - `주문/배송 조회`를 보조 링크 수준으로 하향
  - 인증/대시보드 진입 UI를 공개 헤더에 맞는 저강도 스타일로 조정

### B. 인증 버튼 표현 조정

- `components/auth/AuthButtons.tsx`
  - 공개 헤더에서 사용할 수 있는 저강도 variant 또는 표시 방식 추가 검토
  - 관리자/마이페이지 진입이 과도한 CTA처럼 보이지 않도록 조정

### C. 모바일 메뉴 위계 정리

- `components/common/Header/FullscreenMenu.tsx`
  - 메인 탐색과 보조 유틸 항목의 시각적 구분 강화
  - 필요 시 관리자/주문조회 배치를 하단 보조 영역으로 정리

### D. 문구/테스트 정합성 확인

- 필요 시 `messages/ko.json`, `messages/en.json`의 네비 문구 영향 범위 확인
- 기존 헤더 테스트가 구조 변경으로 깨지지 않는지 확인

## 5) 구현 절차

1. 데스크톱 헤더에서 `작품 구매` 제거 및 우측 액션 영역 단순화
2. `AuthButtons`의 공개 헤더용 표시 강도 조정
3. 모바일 메뉴에서 메인 탐색과 유틸 배치 정리
4. 관련 테스트/번역 영향 범위 점검
5. 변경 결과를 `walkthrough.md`에 기록

## 6) 검증 계획

1. 데스크톱에서 헤더가 다음 구조로 읽히는지 확인
   - 메인 탐색: `전시 작품 / 소개 / 매거진 / 아카이브`
   - 우측 유틸: `검색`
   - 보조 요소: `주문/배송 조회 / 언어 / 관리자`
2. `전시 작품`만으로 작품 목록 진입이 자연스럽게 동작하는지 확인
3. 로그인 상태와 비로그인 상태에서 관리자/마이페이지 진입 UI가 과하게 강조되지 않는지 확인
4. 모바일 메뉴에서 메인 탐색과 보조 항목 구분이 유지되는지 확인
5. `npm run lint`
6. 필요 시 `npm run test -- Header`

## 7) 리스크 및 대응

- `AuthButtons`는 다른 화면에서도 재사용될 수 있어 스타일 변경 범위를 잘못 잡으면 회귀가 생길 수 있다.
  - 대응: 헤더 전용 variant 또는 class 분기로 한정한다.
- `작품 구매` CTA 제거 후 홈/아카이브 등 다른 화면에서 구매 유도가 약해 보일 수 있다.
  - 대응: 헤더가 아닌 작품 목록/상세 내부 CTA가 충분한지 함께 확인한다.
- 모바일 메뉴 조정 시 접근성 속성이나 기존 닫기 동작이 깨질 수 있다.
  - 대응: 기존 구조를 유지한 채 표현 위계만 조정하는 방향을 우선 적용한다.

## 8) 완료 기준

- 공개 데스크톱 헤더에서 `작품 구매` 버튼이 제거된다.
- `전시 작품`과 중복되는 CTA가 사라져 정보구조가 단순해진다.
- `검색 / 주문·배송 조회 / 관리자 진입 / 언어 전환`의 시각적 우선순위가 메인 메뉴보다 낮아진다.
- 모바일 메뉴에서도 메인 탐색과 보조 유틸의 구분이 유지된다.
- 관련 검증 결과가 `walkthrough.md`에 기록된다.
